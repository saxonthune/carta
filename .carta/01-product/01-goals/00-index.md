# Goals

Mission, principles, vocabulary, and foundational intent. Read this title to orient yourself to what Carta is and why it exists.

## Audience

Read this first if you're new to the project. Return here when a design decision needs arbitration — the principles and mission statement define what Carta values.

## What belongs here

- Mission statement and core goals
- Design principles that guide trade-off decisions
- Canonical vocabulary (glossary of domain terms)
- Primary sources — the author's original writings that shaped Carta's direction

## What does NOT belong here

- How the system is built — that goes in `02-architecture/`
- Feature descriptions — those go in `01-product/02-features/`
- Implementation decisions (ADRs) — those go in `02-architecture/06-decisions/`
