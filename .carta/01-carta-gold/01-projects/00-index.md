# Projects

Projects are objective-driven collections of documents that track a body of work across the main doc titles (doc01.05–doc01.04.02). Product information, design decisions, and architecture docs still live in their canonical locations — projects track the scope, gaps, and research needed to get there.

## What belongs here

- **Scope** — what the project aims to change, what success looks like
- **Gap analysis** — what exists today vs what's needed, cross-referencing doc01.05–doc01.04.02
- **Research** — explorations and design work specific to the project objective

## What does NOT belong here

- Canonical product descriptions — those go in doc01.05
- Design decisions and ADRs — those go in doc01.04
- Architecture docs — those go in doc01.04.03
- Code shapes — those go in doc01.04.02

A project is a staging ground. Insights developed here should eventually land in the main titles.

## Lifecycle

Projects don't move or get archived. When a project exits active development, it remains as a record of the work that was done and the reasoning behind it.
