# Operations

Development and process. How to build, test, deploy, and contribute to Carta.

## Audience

Developers setting up their environment, running tests, or preparing releases. Also the CI system — commands documented here should match what CI runs.

## What belongs here

- Dev environment setup and commands
- Test strategy, commands, and CI configuration
- Build and deployment procedures
- Contribution guidelines and PR process

## What does NOT belong here

- Architecture or design decisions — those go in `02-system/`
- Feature specifications — those go in `03-product/`
- Implementation plans — those go in `todo-tasks/`
