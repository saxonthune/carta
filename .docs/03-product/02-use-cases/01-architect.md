# Software Architect

## Persona

A software architect designing a new system or documenting an existing one. Wants to produce a structured specification that AI tools can consume to generate code.

## Goals

- Visually lay out system components and their relationships
- Define custom construct types that match the project's domain
- Organize components into deployment groups
- Compile the architecture into AI-readable output
- Iterate on the design as requirements evolve

## Features Used

- doc03.01.01.01 (Canvas) — primary workspace
- doc03.01.01.02 (Constructs) — modeling software components
- doc03.01.01.03 (Ports and Connections) — defining relationships
- doc03.01.01.04 (Levels) — separating architectural concerns
- doc03.01.01.06 (Schema Editor) — creating domain-specific construct types
- doc03.01.02.01 (Compilation) — producing AI-consumable output
- doc03.01.02.02 (Import/Export) — saving and sharing designs
