# Codex

Meta-documentation: how the `.carta/` documentation system works.

## Audience

Read this title when you need to understand the docs system itself — how it's organized, how to add or update docs, how AI agents navigate it.

## What belongs here

- Documentation format rules (naming, cross-references, front matter)
- Taxonomy rationale (why titles are structured this way)
- Maintenance procedures (versioning, epochs, deprecation)
- AI retrieval patterns (how agents efficiently find information)

## What does NOT belong here

- Documentation about the project's features or architecture — that goes in `02-system/` or `03-product/`
- Project-specific workflow instructions — those go in `04-operations/`
